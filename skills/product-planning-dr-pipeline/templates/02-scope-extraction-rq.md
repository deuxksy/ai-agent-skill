# 02 Scope Extraction Request

## Scope Brief

- Product definition: [한 줄 정의]
- Problem: [해결할 핵심 문제]
- Target users: [주요 사용자]
- Fixed MVP boundary: [고정된 MVP 경계]
- Benchmark input: [`01-dr-merge.md` 경로]
- Constraints: [일정, 팀, 비용, 규제, 운영, 기존 기술]

## Objective

벤치마킹 병합본에서 제품 문제를 해결하는 데 필요한 기능만 추출한다. 경쟁 제품의 기능
목록을 복제하지 말고 사용자 가치와 제품 경계에 따라 `MVP`, `1차 고도화`, `후순위`,
`제외`로 분류한다.

## Scope Rules

- 제품 정의와 목표 사용자를 판단 기준으로 고정한다.
- MVP는 핵심 사용자 흐름을 끝까지 완료하는 최소 범위다.
- 운영 부담, 의존성, 데이터 요구, 권한 복잡성을 함께 평가한다.
- 화면 또는 기술 아이디어를 기능 필요성의 근거로 사용하지 않는다.
- 미정 항목은 MVP에 자동 포함하지 않는다.

## Output Requirements

- 모든 MVP 기능에 사용자 문제, 포함 이유, 수용 기준을 연결한다.
- 모든 제외 항목에 제외 이유를 기록한다.
- 후순위 기능을 MVP 사용자 흐름에 포함하지 않는다.
- 다음 화면설계 단계가 사용할 MVP 기능 목록과 여정을 마지막에 요약한다.

## Handoff to Screen Design

- 확정 MVP 기능 ID 목록
- 핵심 사용자 여정
- 권한 또는 역할 가정
- 주요 오류/예외 시나리오
- 화면설계 전에 검증해야 할 미정 사항

## Expected Merge Structure

`02-dr-merge.md`는 다음 구조를 사용한다.

1. 제품 정의와 고정 MVP 경계
2. 기준본, 보강본, 선택 근거
3. 통합 기능 인벤토리와 분류
4. 확정 MVP 기능 및 수용 기준
5. 후순위, 제외, 충돌, 불확실성
6. Screen Design에 전달할 기능 및 여정 요약
